What's Groovy

  • Dynamic, compiled language for the Java Virtual Machine(JVM).
  • Interoperable with existing Java libraries
  • Concise, easy to read code
  • Used by many major companies including:
Groovy is the main focus of today’s session and the foundation of the Gr8 ecosystem of technologies. To start, Groovy is a dynamic compiled language for the JVM. That mean’s it’s dynamic like languages you may know like Python and PHP, but compiled for efficiency to run on the Java Virtual Machine aka the JVM. Many of you may be asking, why learn yet another programming language? And granted, Groovy is not the best language for every situation. However, in a setting where productivity is paramount and you are looking for the stability of the known architecture stack like Java, Groovy is the best choice. As you’ll see, Groovy’s easy to learn, flexible syntax and interoperability with existing Java libraries, make it a great language to add to your repertoire. It's not just a prototyping solution and many companies use Groovy for enterprise solutions including Netflix, LinkedIn, and Mutual of Omaha.
  • Open source -- maintained by Pivotal and programmers like you!
  • http://groovy-lang.org
  • contribute: https://github.com/groovy/groovy-core
Groovy is also open source. The repository for core is on github and anyone can submit a PR. It's under active development and the community is pretty friendly and welcoming to newcomers.

Once you learn the Groovy language, you are open to a whole new world of technologies including: Grails (the most common web framework in the Groovy eco-system and what many confuse with Groovy itself.) Griffon a desktop framework Gradle a popular build tool that can even be used for building android applications Spock and Geb testing frame,works Ratpack up and coming lightweight web framework (used for this presentation)

Groovy Architecture

You may be wondering how all of this fits together.(explain graphic) An important thing to note is that Groovy is compiled into bytecode for the JVM. Groovy is NOT compiled into Java. This is how groovy can be used in so many different applications. Also notice how a framework like Grails can sit on top of and pull in existing Java libraries and frameworks like Spring and Hibernate.

When to use Groovy

  • existing java libraries
  • developers know java or another JVM language
  • quick prototyping
  • scripting
  • writing tests

When NOT to use Groovy

  • lots of legacy code in another language especially Ruby
  • need 'bare metal' level performance

You can do anything you can do in Java in Groovy. In fact, Almost all Java code is valid Groovy code. Groovy, however, usually has a way to do things with less typing than Java. One big thing you’ll notice is that in Groovy, you don’t always have to declare a function and then call it. Since Groovy is a scripting language, you can just put commands outside of any function scope and it is valid Groovy. Semicolons are unnecessary.

Data Types

  • Strings
  • Collections
    • Lists
    • Maps
    • Ranges
  • Closures
  • Def
We will look at a few datatypes in this workshop: In addition to your normal primitives(int, bool, etc), there are Strings, Collections, and Closures. Collections include classes such as Lists and Maps. Groovy Lists correspond with Java ArrayLists and Groovy Maps correspond with Java HashMaps. Closures are pieces of functionality that you can pass around as data. Groovy is optionally typed. When we don’t care what type a variable is or we want mixed typing, we can use def instead of a data type. Def, short for ‘define’, roughly translates to Java Object, the base class for all other classes.

In order to talk about our next data type closures, I should probably start with how functions work in general. Function calls look very similar to other languages. If nothing else, I hope you remember these three things about groovy. Ok it's really just one. Almost everything is optional. Parenthesis around parameters are optional.
